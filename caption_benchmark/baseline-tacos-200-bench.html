Prompt

You are an expert audio captioner.

Listen carefully to the full audio and describe every audible event.

Output one event per line using exactly this format:

[start_time:end_time] event description

Rules:
- Use seconds for start_time and end_time.
- Keep events in chronological order by start time.
- Overlapping events are allowed and should be listed separately.
- Each event must have exactly one sound source.
- Do not write a global summary.
- Do not invent context that is not audible or strongly implied by sound.
- If all annotated event regions were removed, the remaining audio should be silent.

Each event description should naturally include:
- Source: who or what makes the sound.
- Action: what the source does.
- Descriptor: the sound quality.
- Temporal: how the sound changes over time.
- Context: the audible environment when present.

Good examples:
[0.0:2.6] A train approaches while sounding a loud horn.
[2.6:20.8] A train passes by with a rumbling sound that gradually moves away.
[7.2:8.8] A woman whispers softly in a quiet indoor space.
